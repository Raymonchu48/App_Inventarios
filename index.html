Resumen de seguridad

  • Solo admin puede cambiar roles y borrar productos.
  • editor puede crear y actualizar productos y registrar movimientos.
  • viewer tiene acceso solo de lectura.
  • Los permisos se aplican en interfaz y en RLS dentro de Supabase.
Si eres el primer usuario del sistema, pulsa “Convertirme en admin inicial”. Después desaparece.

Guía rápida

  1. Ejecuta schema.sql y luego seed.sql en Supabase.
  2. Crea tu usuario desde la pantalla de acceso.
  3. Si eres el primero, conviértete en admin inicial.
  4. Desde Administración asigna lectores o editores.
  5. Sube la carpeta a GitHub Pages y empieza a usarla.
